# MergeUi Pattern Library v2 — 디자인 DNA 카탈로그

> **작성일**: 2026-04-22
> **작성**: B(디자인) + F(마케팅) + A(PM) 합동
> **목적**: 레퍼런스 24장을 분해해 재료화한 뒤, MergeUi 고유 창의로 재조합하기 위한 설계 자산
> **규칙**: 한 테마당 기본 패턴 2~3개 + 진화 조합 1개 + 창의 시그니처 1개 필수

## 📐 SECTION 2. 기본 패턴 20종 (응용 단계)

### 🗂 레이아웃 패턴 (L01~L05)

#### L01. 3-column 레이아웃
```
┌──┬──────────────┬──┐
│좌│  메인 콘텐츠  │우│
│네│              │패│
│비│              │널│
└──┴──────────────┴──┘
```
- **구성**: 좌 네비(80~240px) + 메인(가변) + 우 패널(280~360px)
- **우 패널 용도**: 최근 활동 / 알림 / 자산 요약 / 팀 멤버
- **출처**: X Wallet, Belanjo, Catalog, Payflow, DWISON
- **적용 추천**: 크립토/금융/SaaS Overview
- **주의**: 모바일에서는 우 패널이 하단 탭 또는 시트로 내려감

#### L02. Bento Box Grid
```
┌────┬──┐
│ 큰 │중│
├─┬──┼──┤
│소│ 큰 │
└─┴────┘
```
- **구성**: 카드 크기 3~4단계 (1×1, 1×2, 2×1, 2×2) 혼합
- **핵심**: 시각적 리듬. 모든 카드 같은 크기 금지
- **출처**: Panze Project, Stats Dashboard, Pharmly
- **적용 추천**: Project Management / BI / Overview
- **주의**: 모바일에서는 우선순위 순으로 세로 스택

#### L03. Hero-First
```
┌────────────────┐
│   HERO KPI     │ ← 상단 40~50% 점유
├──┬──┬──┬──────┤
│  │  │  │ 기본  │
│  │  │  │ KPI  │
└──┴──┴──┴──────┘
```
- **구성**: 상단 절반을 대형 Hero KPI 하나가 차지
- **출처**: Belanjo(보라 balance), Catalog(campaign), HRD(급여 기간)
- **적용 추천**: E-commerce / Subscription / SaaS 전환 지향
- **효과**: 가장 중요한 메트릭 하나를 극대화 → 스토리 중심 대시보드

#### L04. Single-Chart Dominant
```
┌────────────────┐
│                │
│ ██ 대형 차트 ██│
│                │
├────┬────┬─────┤
│Side│Side│ Side│
└────┴────┴─────┘
```
- **구성**: 화면의 60% 이상을 단일 차트가 점유
- **출처**: Coin.Trade, Crash Game, Crypto Wallet
- **적용 추천**: Trading / Analytics / Realtime Monitoring

#### L05. Perspective / Tilted View
- **구성**: 대시보드 컨테이너를 3D transform으로 약간 기울여 프리뷰 느낌
- **출처**: AI Agent, Panze Project(상단 프로필), CoinSphere
- **적용 추천**: 랜딩 페이지 히어로 섹션 / 썸네일 특화
- **주의**: 실제 사용 화면에는 쓰지 않고 **썸네일/마케팅 이미지에만**

### 🃏 카드 패턴 (C06~C10)

#### C06. Gradient Hero Card
- **구성**: 대형 컬러 블록(보라/녹/오렌지/다크) + 메인 숫자 + 보조 텍스트 + CTA 버튼
- **출처**: Belanjo, Catalog, Financial Dashboard, HRD, Alice, Pharmly upgrade 박스
- **적용**: 잔액/매출/가입자 등 **"제일 중요한 하나"** 강조

#### C07. Donut Center Overlay
- **구성**: 도넛 차트 중앙에 큰 숫자 + 작은 라벨 (cutout 60~70%)
- **출처**: Shopr(139% Total), Chef Control, Financial Dashboard(78%)
- **적용**: 비율/진행률 단일 메트릭

#### C08. Multi-Color Icon Tiles (KPI)
- **구성**: KPI 4~8개를 각자 다른 컬러/아이콘 타일로
- **출처**: Pharmly(8종), Alice(오렌지), Desklog(보라 강조)
- **주의**: 컬러 과다 주의 — 한 색 hue 안에서 명도 변화로

#### C09. Progress Bar List
- **구성**: 항목 이름 + 값 + 컬러 진행바 (인보이스/예산/상태)
- **출처**: Panze Project(invoice), Chef Control, Fitness(overview)
- **적용**: 상태 기반 리스트 (결제 상태, 목표 달성률)

#### C10. Inline Tooltip Card (차트 위)
- **구성**: 라인/바 차트 위에 호버 시 뜨는 데이터 카드 (정적 버전)
- **출처**: Belanjo($42,000 툴팁), Desklog, Panze Purple(포인터), Datio
- **적용**: 라인/바 차트의 포커스 포인트 강조

### 📊 차트 패턴 (G11~G15)

#### G11. Dual Line + Area Fill
- **구성**: 2개 라인 + 한쪽만 영역 채우기 (그라디언트 페이드)
- **출처**: Belanjo, Orange Bank, Datio
- **효과**: 시간 시리즈 비교 + 시각적 풍성함

#### G12. Semi-Circle Gauge (대형)
- **구성**: 반원 arc + 중앙 숫자 + 컬러 세그먼트
- **출처**: CoinSphere(7.52%), Chef Control(5,868), Stats(90/25c)
- **적용**: 목표 달성률, 시스템 상태

#### G13. Heatmap Grid
- **구성**: N×M 타일 컬러 강도 그리드
- **출처**: Crypto Wallet(activity), Security(threat tactics)
- **적용**: 시간대별 활동, 위험도 매트릭스

#### G14. Stacked Bar Colorful
- **구성**: 스택 바 + 세그먼트 컬러 5~7가지
- **출처**: Security, Catalog(engagement analytics)
- **적용**: 카테고리 믹스 + 시간 추이 동시

#### G15. Dense Sparkline Grid
- **구성**: 미니 라인 6~10개를 격자로
- **출처**: 내부 제작(td_v1 watchlist), 내부 추가(bw_v1)
- **적용**: 다중 KPI 트렌드 요약

## 🔀 SECTION 3. 진화 조합 10종 (중급 — 레퍼런스에 없는 믹스)

### E01. 뉴모피즘 + 촛불차트
- 기존: 촛불=플랫 다크 / 뉴모=소프트 밝음
- 진화: 뉴모 카드 안에 촛불차트 → **부드러운 트레이딩 대시보드**
- 적용: 새 테마 `ns_v1` (Neo-Stock)

### E02. 브루탈리즘 + 벤토 박스
- 기존: 브루탈=단조/굵은 / 벤토=화려/다양
- 진화: 굵은 테두리 + 두꺼운 타이포로 벤토 박스 구성
- 적용: bd_v1 리빌드

### E03. 픽셀 폰트 + Hero 카드 + 글래스모피즘
- 기존: 픽셀은 레트로 / 글래스는 모던
- 진화: 투명 유리 위 픽셀 타이포 Hero → **RetroFuturism**
- 적용: 새 테마 `rf_v1`

### E04. 글래스모피즘 + 3D 일러
- 기존: 이미 gl_v1 있지만 일러 없음
- 진화: gl_v1에 3D 캐릭터 1개 추가 → 친근함 + 프리미엄 동시
- 적용: gl_v1 리빌드

### E05. 클레이모피즘 + 도넛 오버레이 + Hero
- 기존: 클레이=말랑 / 오버레이=플랫
- 진화: 말랑한 도넛 + 중앙 큰 숫자 + 그림자 깊이
- 적용: fi_v1 리빌드

### E06. CRT 터미널 + 실시간 스트림 게이지
- 기존: crt_v1 정적 / 반원게이지 정적
- 진화: ASCII 흐르는 반원게이지 + 스캔라인 위 실시간 숫자
- 적용: crt_v1 강화

### E07. 페이퍼 텍스처 + 히트맵 + 잉크 마크
- 기존: 종이 = 정적 / 히트맵 = 디지털
- 진화: 크래프트 종이 위 잉크 도장 처럼 찍힌 히트맵
- 적용: 새 테마 `pp_v1` (Paper Planner)

### E08. 다크 네온 + 진행바 리스트 + 3-column
- 기존: 네온 = 장식 / 3-column = 구조
- 진화: 좌측 네온 사이드바 + 우측 알림 피드 + 진행바 네온 블리딩
- 적용: 새 테마 `nx_v1` (Neon X)

### E09. 럭셔리 골드 + 세미서클 게이지 + 큰 타이포
- 기존: 럭셔리 = 여백 / 게이지 = 차트
- 진화: 대형 게이지 + 얇은 금선 + 세리프 대비
- 적용: lx_v1 리빌드

### E10. 모노 1px 헤어라인 + 스파크라인 밀집 + 타임라인
- 기존: 모노 = 제한적 / 밀집 = 복잡
- 진화: 최소한의 컬러 + 조밀한 정보 밀도 → "데이터 신문"
- 적용: ln_v1 리빌드

## 💡 SECTION 4. 창의 발명 10종 (고급 — MergeUi 시그니처)

**경쟁사(Tailwind UI, shadcn, Creative Tim)도 안 하는 것들**

### X01. 🕐 시간대 자동 변환 (Temporal Theme)
- **아이디어**: 사용자의 현지 시간대에 따라 컬러 톤 자동 변화
  - 오전(06~11): 밝은 파스텔
  - 오후(12~17): 중간 톤
  - 저녁(18~22): 따뜻한 앰버
  - 심야(23~05): 다크 모드 + 약한 블루
- **구현**: CSS variables + `prefers-color-scheme` + JS 시간 훅
- **차별화**: 판매 시 "시간 따라 숨 쉬는 대시보드" 슬로건
- **대상**: 새 테마 `tm_v1` 또는 기존 sd_v1 진화

### X02. 📖 LNB 스토리 모드 (Sidebar Narrative)
- **아이디어**: 좌측 네비가 단순 링크가 아니라 **진행형 스토리**로 표현
  - "오늘 당신의 비즈니스는..." → 각 섹션에 현재 상태 뱃지
  - 예: `📊 Deals (3 new)` / `💰 Revenue (↑12%)` / `👥 Team (2 offline)`
- **차별화**: 네비가 "링크 모음"에서 "대시보드 요약"으로 진화
- **대상**: cr_v1 (CRM) 리빌드

### X03. 🎛 타자기 터미널 인터랙션 (Typewriter CLI)
- **아이디어**: crt_v1 확장 — 실제 명령어 입력 가능
  - `stats:cpu` → CPU 위젯 확대
  - `deploy:status` → 배포 히스토리 표시
  - `help` → 사용 가능 명령어 리스트
- **차별화**: 키보드 유저 전용 대시보드 조작
- **대상**: crt_v1 리빌드

### X04. 🧲 물리 엔진 드래그 위젯 (Physics-based Dock)
- **아이디어**: 위젯 드래그 시 자석 스냅 + 약간의 물리 피드백
  - 위젯을 드래그하면 살짝 기울어짐 (스프링)
  - 놓으면 그리드 슬롯에 스냅 (딸깍 피드백)
- **차별화**: 대부분 대시보드는 그냥 이동만 → 피드백 추가로 "프리미엄감"
- **대상**: bi_v1 리빌드

### X05. 📸 ASCII 모드 토글 (Heritage Toggle)
- **아이디어**: 한 버튼으로 모든 차트/KPI가 **ASCII 버전**으로 전환
  - 정상 모드: 일반 차트
  - ASCII 모드: `▂▄▆█▇▅▃▁` 블록 문자로 그린 차트
- **차별화**: 엔지니어 감성 + 이스터에그 + 복사 가능
- **대상**: bw_v1 (브루탈 모노) 리빌드

### X06. 🎬 스토리텔링 요약 뷰 (Auto-narrate)
- **아이디어**: 대시보드 상단에 "이번 주 요약" 자동 생성 텍스트
  - "이번 주 매출은 지난 주 대비 12% 증가했습니다. 가장 큰 기여자는 엔터프라이즈 플랜 3건입니다."
  - 정적 템플릿 기반(클라이언트에서 값만 치환) — AI 실제 호출 아님
- **차별화**: "데이터 → 언어" 자동 변환 프리뷰 (나중에 진짜 AI 연결 가능)
- **대상**: bi_v1 리빌드

### X07. 🔀 듀얼 비교 뷰 (Split Compare)
- **아이디어**: 한 화면에 "지난 분기 vs 이번 분기" 50:50 분할
  - 좌우에 동일 레이아웃 + 다른 기간 데이터
  - 중앙 슬라이더로 비율 조절 가능
- **차별화**: 비교 분석 대시보드는 많지만 "같은 UI 두 벌 쌍쌍 비교"는 드묾
- **대상**: 새 테마 `dv_v1` (Dual View) 또는 ec_v1 확장

### X08. 🔊 Soundscape Feedback (사운드 UI)
- **아이디어**: 이벤트마다 미세한 사운드 피드백 (ON/OFF 가능)
  - 알림 도착 → 딩
  - 매출 상승 → 띵 (오름조)
  - 서버 에러 → 저음
- **차별화**: 소리 없는 대시보드가 당연한 시대에 역행 → 엔지니어/크리에이터층 호응
- **대상**: fi_v1 또는 td_v1 (Trading)

### X09. ⏪ 시간축 스크러빙 (History Scrubber)
- **아이디어**: 모든 데이터에 시간 슬라이더 "되감기" 가능
  - 상단 슬라이더를 과거 시점으로 드래그 → 모든 차트/KPI가 그 시점 값으로
  - 비디오 편집기 스크러빙과 유사
- **차별화**: "과거를 걸어 다니는 대시보드" — 분석가 감동 포인트
- **대상**: bi_v1 또는 새 테마

### X10. 📱 햅틱 레이아웃 모프 (Responsive Morph)
- **아이디어**: 화면 크기 변화 시 **컴포넌트가 타입 자체를 변형**
  - 데스크톱: 도넛 차트
  - 태블릿: 진행바 리스트
  - 모바일: 단일 큰 숫자 + 스파크
- **차별화**: 대부분 "리플로우"만 함 → "변신"은 거의 없음
- **대상**: 모든 테마 공통 (기술 시그니처)

### 🥈 2. ec_v1 — Triple Column E-Commerce
- **레이아웃**: L01 3-column (좌 네비 + 중 매출/주문 + 우 고객 패널)

### 🥉 3. cr_v1 — Narrative CRM

## 📏 SECTION 7. 적용 룰 (B가 제작 시 반드시 지킬 것)

1. **한 레퍼런스에서 최대 2개 요소만 차용** — 복제 방지
2. **최소 3개 다른 레퍼런스 조합** — 독창성 보장
3. **패턴 2~3개 + 진화 1개 + 창의 1개 필수** — 차별성 보장
4. **컬러/타이포/여백은 MergeUi 브랜드 DNA로 재작성**
5. **최종 결과물이 레퍼런스 24장 중 어느 것과도 한눈에 닮지 않게**
6. **모바일 반응형 동시 설계** — 깨지면 팔리지 않음
7. **썸네일이 시그니처 요소를 즉시 보여줘야** — 일반 아이콘 금지
