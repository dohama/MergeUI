# MergeUi 상품 제작 표준 가이드

> 모든 테마·컴포넌트 제작 시 반드시 참고하는 문서
> 최종 수정: 2026-04-18

---

## 1. 기본 원칙

- **우리가 파는 건 디자인 템플릿이다.** 디자인이 곧 상품이므로 퀄리티가 사업의 생사를 결정한다.
- **"와, 이런 서비스가 있었구나"** 수준의 고퀄리티가 기준이다. 그냥 괜찮은 수준은 불합격.
- **화면만 꾸미는 게 아니다.** 실제 SaaS/CRM/E-Commerce 등에서 사용하는 데이터와 항목 기반으로 구성한다.
- **개발자가 가져다 바로 쓸 수 있어야 한다.** 5분 내에 본인 데이터에 머지 가능한 구조.

---

## 2. 컬러 전략

### 규칙
- 각 테마별 **배경 톤 1개 고정**. 라이트/다크 전환 기능 넣지 않는다.
- **액센트 컬러 프리셋**을 유동적으로 제공 (테마별 3~5벌).
- 무료 템플릿은 **프리셋 1개 고정**.
- 구독자는 tokens.css 수정으로 자유 커스터마이징 가능 (원래 되는 것).

### 주의사항
- **컬러 다양성.** AI가 기본으로 보라색을 쓰는 경향이 있으니, 항상 다양한 컬러를 캡틴에게 제안할 것. 보라색도 좋지만 매번 보라색이면 안 된다.
- 메인 컬러가 텍스트로 사용될 때 **배경 대비 가독성** 반드시 확인 (WCAG AA 4.5:1).
- 같은 컬러로 도배하면 눈아프다. **메인/서브/서드 밸런스** 필수.
- 서브 컬러가 밝으면 본문에서 자제, 그라디언트나 포인트에만 사용.

### SoftDesk 예시
- 메인: #1ACEFF
- 서브: #44EEEF (그라디언트용, 밝은 곳 자제)
- 서드: #6C5CE7 (밸런스용)

## 3. 폰트

### 규칙
- **저작권 무료 폰트만 사용** (Google Fonts OFL 라이선스).
- 작업 시작 전 디자이너(B)가 **분야에 맞는 폰트를 캡틴에게 제안**할 것.
- "Inter만 쓰면 됩니다"는 금지. 테마 컨셉에 맞는 폰트를 찾아서 제안.

### 추천 폰트 풀
| 폰트 | 특징 | 잘 어울리는 테마 |
|------|------|----------------|
| Plus Jakarta Sans | 따뜻하고 부드러움 | 뉴모피즘, 친근한 SaaS |
| DM Sans | 모던 지오메트릭 | 미니멀, 핀테크 |
| Inter | 가장 무난, 가독성 좋음 | 범용 |
| Manrope | 기하학적 + 읽기 편함 | 데이터 중심 |
| Outfit | 깨끗하고 현대적 | 트렌디한 SaaS |

## 5. 코드 구조 — 개발자 친화적

### 데이터 분리 (필수)
```
templates/테마명/
├── index.html           # 메인 페이지
├── css/
│   ├── 테마-tokens.css  # 커스터마이징 포인트 (컬러, 폰트, 간격)
│   └── 테마.css         # 전체 스타일
├── js/
│   ├── 테마.js          # 인터랙션
│   └── data.js          # ★ 더미 데이터 분리 — 이 파일만 수정하면 데이터 교체
└── README.md            # 설치/커스터마이징 가이드
```

### data.js 예시
```javascript
// ============================================
// SoftDesk — Sample Data
// 이 파일의 값을 수정하면 대시보드 데이터가 바뀝니다.
// ============================================

  kpi: [
    { label: 'Total Users', value: '2,847', trend: '+12.5%', trendDir: 'up' },
    { label: 'Monthly Revenue', value: '$18.2k', trend: '+8.3%', trendDir: 'up' },
    { label: 'Active Rate', value: '94.2%', trend: '+2.1%', trendDir: 'up' },
    { label: 'Churn Rate', value: '1.8%', trend: '-0.4%', trendDir: 'down' },
  ],
  revenueChart: {
    labels: ['Jan','Feb','Mar','Apr','May','Jun','Jul','Aug','Sep','Oct','Nov','Dec'],
    data: [8200, 9100, 8800, 10200, 11500, 12100, 13400, 14200, 15100, 16300, 17500, 18200]
  },
  recentOrders: [
    { name: 'Sarah Chen', plan: 'Pro', amount: '$19.00', status: 'Paid', date: 'Apr 17, 2026' },
    // ...
  ]

```

### 코드 주석 규칙
```html
<!-- ============================================
     KPI Cards Section
     - 4개 카드: tokens.css의 --sd-primary 등으로 아이콘 색상 변경
     - data.js의 SOFTDESK_DATA.kpi 배열로 데이터 변경
     ============================================ -->
```

### tokens.css 주석 규칙
```css
/* ===== 이 파일만 수정하면 전체 테마가 바뀝니다 =====
   1. 컬러 변경: --sd-primary, --sd-secondary 값 수정
   2. 폰트 변경: --sd-font 값 수정
   3. 간격 변경: --sd-space-* 값 수정
   4. 둥글기 변경: --sd-radius-* 값 수정
   ================================================= */
```

## 6. MergeUi 독보적 요소

### 경쟁사와 차별화 포인트

| 요소 | 설명 |
|------|------|
| **tokens.css 원파일 커스터마이징** | 파일 하나만 수정하면 전체 테마가 바뀜. 가이드 포함 |
| **data.js 데이터 분리** | HTML 안 건드리고 데이터만 교체 가능 |
| **컴포넌트 독립 복사** | KPI 카드, 차트, 테이블을 개별로 복사해서 다른 프로젝트에 붙여넣기 가능 |
| **뉴모피즘 유틸리티 클래스** | `.neu-raised`, `.neu-inset`, `.neu-pressed` — 어디든 적용 가능 |
| **실제 SaaS 시나리오 더미** | 가져다 쓰면 바로 프로토타입 가능한 현실적 데이터 |
| **액센트 프리셋** | 주석 해제만으로 전체 테마 컬러 변경 |
| **코드 주석 가이드** | "여기를 수정하면 OO이 바뀝니다" 식의 친절한 안내 |

## 7. 보안 — 코드 보호

### 홈페이지에서 보여줄 때
- 테마 프리뷰는 **iframe**으로 보여줌 (경쟁사 표준)
- iframe 내 코드에 **난독화(obfuscation)** 적용 검토
- **우클릭 방지, 드래그 방지**는 경험 해치므로 하지 않음
- 핵심 방어: "50개 파일을 하나하나 복사하는 것보다 $19 내고 zip 받는 게 빠르다"는 가치 제공

### 다운로드 패키지
- Pro 사용자만 다운로드 가능 (서버에서 권한 확인)
- 다운로드 시 라이선스 키 포함
- 코드에 라이선스 주석 포함: `/* Licensed to: user@email.com | Key: MERGE-PRO-XXXX */`

## 8. 테마별 필수 확인 — 실제 사용 데이터

### SaaS 대시보드 필수 항목
- KPI: MRR, ARR, Active Users, Churn Rate, LTV
- 차트: 매출 추이, 가입자 추이, 플랜별 분포
- 테이블: 최근 가입자, 최근 결제, 티켓/문의
- 페이지: Overview, Analytics, Users, Billing, Settings

### CRM 대시보드 필수 항목
- KPI: 총 연락처, 활성 거래, 성사율, 파이프라인 가치
- 차트: 파이프라인 퍼널, 월별 성사율, 거래 단계별 분포
- 테이블: 최근 거래, 연락처 목록, 활동 로그
- 페이지: Pipeline, Contacts, Deals, Activities, Reports

### E-Commerce 대시보드 필수 항목
- KPI: 총 매출, 주문 수, 평균 주문가, 반품률
- 차트: 매출 추이, 카테고리별 매출, 지역별 분포
- 테이블: 최근 주문, 인기 상품, 재고 현황
- 페이지: Overview, Orders, Products, Inventory, Customers

## 9. 디자이너(B) 특별 지침

1. **작업 시작 전 반드시 캡틴에게 제안**할 것:
   - 이 테마에 이 폰트가 어울릴 것 같습니다 (이유 포함)
   - 이 컬러 조합을 제안합니다 (메인/서브/서드 + 샘플 이미지)
2. **뻔하지 않게.** 재미있는 요소를 넣을 것.
3. **의견 적극 제시.** 캡틴이 허용했음. 좋은 아이디어가 있으면 말할 것.
4. **검수 시 1px도 놓치지 않을 것.** 텍스트 가독성, 컬러 밸런스, 간격 일관성.

## 10. 전 에이전트 공통 지침

- 모든 테마 작업 마무리 시 **6명 전원 합동 검수** 필수
- 검수 시 **"이걸 보고 내가 $19 낼 의향이 있는가?"** 를 기준으로 판단
- 심각도는 **1~5 수치**로 표기 (5=즉시수정, 1=안해도됨)
- SoftDesk가 **모든 테마의 기준**. 여기서 확립된 퀄리티/구조를 이후 테마에 적용

## 11. 작업 시 반복 실수 방지 — 절대 놓치지 말 것

> 실제 작업에서 캡틴이 지적한 사항들. 한 번 지적된 것은 두 번 다시 실수하지 않는다.

### 가독성
- [ ] LNB 메뉴 텍스트가 배경과 구분되는가? (뉴모피즘 배경은 밝으므로 --text-muted 사용 금지, --text-secondary 이상)
- [ ] 메인 컬러를 텍스트로 사용할 때 배경 대비 가독성 확인
- [ ] 차트 라벨, 범례 텍스트가 보이는가?

### 차트/그래프
- [ ] 모든 차트가 실제로 렌더링되는가? (빈 화면 절대 불가)
- [ ] Chart.js 색상을 CSS 변수에만 의존하지 말고 fallback 직접 지정
- [ ] 차트 데이터는 data.js에서 가져오기 (하드코딩 금지)
- [ ] 도넛/파이 차트 범례와 실제 차트 색상 일치 확인

### 컬러 밸런스
- [ ] 같은 페이지에 동일 컬러가 3개 이상 반복되지 않는가?
- [ ] 메인/서브/서드 컬러가 골고루 사용되었는가?
- [ ] 프로젝트 카드, KPI 카드 등 반복 요소에서 상태별로 다른 컬러 사용

### 불필요한 요소
- [ ] 의미 없는 버튼 없는가? (햄버거 버튼 등 기능 없으면 제거)
- [ ] 다크모드 토글 없는가? (캡틴 결정: 배경 고정)
- [ ] HTML에 쓰이지 않는 그룹 라벨이나 빈 div 없는가?

### 데이터 일관성
- [ ] 더미 데이터가 MergeUi 전용이 아닌 범용 SaaS 데이터인가?
- [ ] 사이드바 메뉴 순서와 모바일 네비 순서가 동일한가?
- [ ] 5개 페이지 사이드바 HTML이 동일한가? (active 클래스만 다르게)

### 코드 품질
- [ ] data.js로 데이터 분리되었는가?
- [ ] tokens.css에 커스터마이징 가이드 주석이 있는가?
- [ ] HTML 섹션마다 주석이 있는가? ("데이터 변경: data.js → OO")
- [ ] README.md가 있는가?
- [ ] 뉴모피즘 유틸리티 클래스(.neu-raised 등)가 있는가?
- [ ] 하드코딩된 색상값이 없는가? (CSS 변수 또는 직접 지정 변수 사용)

## 12. 디자인 시스템 — 테마별 필수 정의

> 모든 테마/컴포넌트 제작 전에 디자인 시스템을 먼저 정의한다.

### 디자인 시스템 문서 필수 항목

| 항목 | 설명 | 파일 |
|------|------|------|
| 컬러 팔레트 | 메인/서브/서드/상태색 + HEX + 용도 | design/themes/테마명-design-system.md |
| 타이포그래피 | 폰트명, 크기 스케일, 굵기, 행간 | 위와 동일 |
| 간격 스케일 | 4px~64px 8px 그리드 | tokens.css에 정의 |
| 그림자 체계 | raised/inset/pressed 단계별 | tokens.css에 정의 |
| Border Radius | 요소별 둥글기 | tokens.css에 정의 |
| 컴포넌트 스펙 | KPI 카드, 차트, 테이블 등 치수/패딩/간격 | design-system.md |
| 반응형 브레이크포인트 | 데스크톱/태블릿/모바일 동작 | design-system.md |

### 작업 순서
1. **디자인 시스템 정의** (B 디자이너 주도)
2. **캡틴 승인**
3. **tokens.css 생성**
4. **HTML/CSS 구현**
5. **data.js 데이터 분리**
6. **전 에이전트 합동 검수**
